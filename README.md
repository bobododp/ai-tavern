# AI 酒馆

一个面向个人创作的本地 AI 剧情与角色扮演桌面应用。它把「写故事、接入模型、继续对话、生成画面」收拢到一个轻量、直观的工作流里：不用面对复杂的参数面板，填入自己的 API 配置后即可开始创作。

> 当前为 v1.0.0 原型首版，优先保证本地可用、数据可控与创作流程顺畅。

![AI 酒馆界面](assets/tavern-scene.svg)

## 为什么做它

许多 AI 角色扮演工具功能很强，但设置链条长、概念多。AI 酒馆更关注一件事：让创作者专注于故事本身。

- **自己的模型，自己选择**：可接入兼容 OpenAI 风格接口的聊天模型与生图引擎，不绑定特定服务商。
- **故事不是一次性对话**：按故事档案管理角色、设定、进度和关键记忆，适合持续创作。
- **从文字到画面自然衔接**：先整理绘画提示词、由用户确认后再生成；画面自动归入当前故事与画廊。
- **复杂能力藏在合适的位置**：上下文预览、压缩摘要、世界书导入等能力都保留，但不让它们打断日常写作。
- **数据留在本地**：故事、设置和界面状态默认保存在本机；项目不内置任何个人 API 地址或密钥。

## 功能一览

### 创作与叙事

- 多故事创建、切换、继续编辑与删除
- 行动、对话、描写、观察 / 生图等输入方式
- 故事级设置：叙事规则、回复长度、自动摘要与下一步引导
- 上下文预览与压缩摘要，帮助长故事保留人物、物品和事件线索
- 基础故事模板，开箱即可体验

### 世界与素材

- 世界书导入、预览与写入流程
- 故事封面生成：先确认提示词，再提交生成
- 画廊查看大图、删除图片、打开本地图片目录
- 图片与故事上下文关联，便于回看创作过程

### 模型与桌面体验

- 聊天模型使用兼容 OpenAI 风格的接口配置
- 生图模型可自行填写 API 地址、模型名与密钥
- 本地生图代理，降低浏览器跨域限制带来的使用门槛
- Electron Windows 桌面应用，开发态与打包态均可运行
- 亮 / 暗主题与后续皮肤系统扩展位

## 快速开始

### 方式一：直接运行桌面应用

在发布包中运行安装版或便携版程序。首次打开后，到「设置」中填入自己的聊天模型和生图模型配置。

### 方式二：从源码启动

环境要求：Node.js 18 或更高版本。

```bash
npm install
npm run app:dev
```

如需以浏览器方式体验静态页面，可双击 `启动酒馆.bat`。它会启动本地图片代理并打开页面。

### 基础检查

```bash
npm run check
npm run check:scripts
```

## 隐私与使用边界

- API Key 只由用户在本地填写；仓库不会包含个人密钥、历史故事或已生成图片。
- 故事与界面数据默认保存在浏览器 `localStorage` 或 Electron 的本地用户数据目录。
- 调用模型时，请自行了解并遵守所选服务商的计费、内容和隐私规则。
- 本项目是本地创作工具原型，不提供云同步、账号体系、自动更新或代码签名。

## 项目结构

```text
assets/          图标与界面素材
electron/        Windows 桌面应用主进程与预加载逻辑
scripts/         故事数据、运行时、编辑器、渲染与交互模块
image-proxy.js   本地静态服务与生图代理
index.html       页面结构
styles.css       主题与界面样式
```
