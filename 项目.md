# AI 酒馆项目说明

## 项目定位

这是一个面向长期故事体验的本地 AI 酒馆原型。当前目标是把可用原型稳定下来，完成阶段性收尾，并提供 Windows 桌面安装包。

当前形态：
- 已增加 Electron 桌面壳，可直接运行桌面 App。
- 仍保留静态页面结构，必要时可直接打开 `index.html` 检查。
- 使用 `localStorage` 保存故事、API 设置和界面状态。
- 聊天模型可由前端直接调用。
- 生图默认通过本地代理 `image-proxy.js` 转发，避免浏览器 CORS 拦截。
- 桌面 App 会内置启动代理，并把生成图片保存到 Electron 用户数据目录。

## 核心能力

- 多故事管理：创建、继续、删除、故事设置。
- 长期故事对话：支持行动、对话、描写、观察/生图输入模式。
- 上下文预览与压缩摘要：保留阶段性记忆、关键人物、物品、事件链。
- 世界书导入：支持创建故事时导入、预览、翻译写入。
- 生图链路：先整理绘画提示词，再由用户确认生成；图片进入当前故事上下文，也进入画廊仓库。
- 故事级成熟模式接口：以 `matureMode` 保存，兼容旧字段 `matureUnlocked`。
- UI 皮肤预留：当前通过 CSS 变量和 `light-theme` 控制主题，后续可扩展为 `body[data-skin="..."]` 或独立皮肤 CSS 文件。
- 打包方向已落地为 Electron Windows 桌面 App，内置本地代理和图片目录。
- 皮肤系统方向：先保留 CSS 变量和 `data-skin` 扩展口，不急着重做 UI。

## 重要文件

- `index.html`：页面结构、弹窗、按钮入口。
- `styles.css`：布局、主题、组件样式。
- `app.js`：兼容入口。
- `scripts/core.js`：通用常量、格式化、压缩状态工具、绘画风格与人物锚点工具。
- `scripts/story-data.js`：故事默认结构、样例数据、旧数据修复。
- `scripts/story-storage.js`：本地存储读写。
- `scripts/story-runtime.js`：故事切换、消息写入、生图链路、运行时状态推进。
- `scripts/story-engine.js`：上下文构建、聊天模型调用、回复解析、压缩摘要。
- `scripts/story-editor.js`：创建故事、故事设置、世界书导入。
- `scripts/ui-render.js`：页面渲染。
- `scripts/ui-events.js`：事件绑定。
- `image-proxy.js`：本地静态服务、生图代理、图片保存、打开文件夹。
- `electron/main.js`：Electron 主进程，启动窗口和内置代理。
- `electron/preload.js`：向前端注入桌面代理地址。
- `package.json`：Electron 开发启动与打包配置。
- `dist/`：安装包输出目录，不纳入源码交付。
- `启动酒馆.bat`：启动本地代理并打开页面。
- `验收清单.md`：收尾前手动验收步骤。
- `问题清单.md`：冻结前剩余风险。

## 本地数据键

- `tavern-stories`
- `tavern-active-story-id`
- `tavern-api-settings`
- `tavern-theme`
- `tavern-sidebar-collapsed`

不要随意改这些键名；如需调整，必须做兼容迁移。

## 当前收尾原则

1. 先修阻断、乱码、空按钮、重复事件。
2. 保持现有原型可直接打开、可继续故事、可使用生图代理。
3. 不引入数据库、新正式后端或账号体系。
4. 皮肤重做、自动更新、代码签名、正式云端存储进入下一阶段。
